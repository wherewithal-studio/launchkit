1. Executive Summary

P0 — STRUCTURAL Launchkit has no product yet. The deployed public/index.html is 56 lines: one headline, one sentence, "Content coming soon." There is no information architecture, no navigation, no content, no component system. This scan functions as a design brief and capability roadmap, not an audit of existing work.
P1 — CARRIED 7th cycle Cold-start identity undefined. The product has not decided what it is: (a) a marketing launch-page hub, (b) an internal launch checklist and SOP system, or (c) a brand documentation portal. Without that decision, no design work can be validated.
P1 — CARRIED 7th cycle Output clarity still missing. No surface tells a visitor what Launchkit produces. Prior recommendation — "Launchkit produces a brand narrative package — one brief, one visual system, zero ambiguity" — remains unimplemented.

Killer feature: The shareable brand brief URL. If launchkit.wherewithal.studio/dodek rendered a complete, public-facing brand package — positioning, colors, type, copy — that single page would simultaneously be the product demo, the onboarding artifact, the press kit, and the design spec. Everything else is secondary to building that.

Single strategic recommendation: Ship a live preview of what a completed Launchkit package looks like, using DODEK or PAM as the example. That preview is the landing page, the product demo, and the trust signal simultaneously. Build it before building any configuration UI.

2. Product Brief

Product purpose

Launchkit is the GTM operating system for Wherewithal ventures. It consolidates positioning, messaging, and visual identity so that every venture ships with a consistent, deployable brand narrative rather than a folder of unresolved decisions.

Business model signal

Internal infrastructure and portfolio tooling. Business value is indirect: Launchkit compresses time-to-proof for every venture, which validates the portfolio thesis. May eventually externalize as a productized service for founders outside Wherewithal.

Primary persona

Founder-Operator building a new venture inside Wherewithal. Needs to go from "concept approved" to "landing page live with correct positioning" in one session. Understands the thesis but hasn't internalized every venture's voice. Needs a system that makes decisions, not a blank canvas.

Secondary persona

External partner or advisor reviewing a venture's positioning. Needs to read a brand narrative package without diving into Notion or a repo. A shareable, self-contained brief page would serve them with no onboarding required.

Success frame

Primary: Founder-Operator opens Launchkit, selects a venture, and has an exportable brand package (hero copy, color system, type system, positioning statement) in under 20 minutes without opening Figma or a blank doc. Secondary: Advisor opens a shared link and reads the venture's brand package in under 2 minutes.

Competitive positioning

Not competing with Figma, Notion, or Webflow. Competing with the chaos of a shared Google Drive folder and a founder doing positioning from memory at 11pm before a launch.

Known constraints

Cloudflare Pages static site. No server-side state. Three font families: Bebas Neue, Barlow Condensed, JetBrains Mono. Build stage — no production users. Prior P0: inline button at 0.65rem (not found in current index.html — likely in infrastructure/public venture-summary page, not launchkit repo itself).

3. Security Flags

P2 — CARRIED 7th cycle — CONFIRMED MISSING Pre-commit hook absent. launchkit/.git/hooks/ contains only sample hooks. Launchkit is the only Wherewithal project without an active secrets-scanning pre-commit hook. No hardcoded secrets found in current source (risk is low while product is a placeholder), but the hook must be in place before any real credentials touch the repo.

No other security findings. No hardcoded credentials, XSS vectors, or auth bypass surfaces exist — the product has no dynamic behavior. Deploy pre-commit hook before adding any API keys or worker configuration.

8. Visual Design Possibilities

Visual Potential Inventory

The design language — dark background, Bebas Neue / Barlow Condensed / JetBrains Mono, accent blues and purples — is a strong foundation. The three-family system creates technical authority hierarchy. The gap is not the language, it's the absence of content to apply it to.

Typography Possibilities

The current h1 at 2.5rem is undersized for a hero. At 4–6rem with tight letter-spacing, the same headline reads as an intentional editorial statement. Barlow Condensed at 0.72rem+ with 0.1em tracking on section labels creates a clear tertiary layer. JetBrains Mono treating positioning statements as "code output" reinforces the compiler metaphor and is tonally consistent with the Wherewithal thesis.

Color Possibilities

Each venture has its own accent. Launchkit's own accent should distinguish it from the ventures it serves — consider a warm metallic like #c8a96e (gold) as the meta-brand color. Carries the connotation of "the kit that holds all the others" without competing with any single venture's palette.

Spatial Possibilities

With minimal current elements, generous spacing (80–120px vertical rhythm) would signal premium quality. The current 2rem body padding is cramped for a hero page. Full-bleed header zone at 80px padding would immediately communicate that this is a considered product.

The Visual Design Bet

Replace the placeholder with a full-viewport hero that shows a real DODEK brand brief card — actual positioning statement, actual color swatches, actual type specimen — rendered in the Launchkit frame. That single change makes the product real without writing a line of application code.

Technical Integrity Flags

Missing No OG / meta tags. <title> is "Launchkit" — no description, no og:image, no og:title. Any share to Slack, Twitter, or email renders a blank card.
Missing No favicon. Browser tab shows default browser icon.
Note No 404 page. Cloudflare Pages default 404 serves for any unknown path.
Note Brand fonts not loaded. Landing page uses -apple-system fallback. Acceptable for placeholder; must be addressed before real content ships.

9. Design Weight Analysis

Design Weight Inventory

Currently: one h1 carries 100% of perceptual weight. That weight is wasted — nothing follows. The second sentence ("Content coming soon") actively destroys trust by confirming the product is incomplete.

Killer Feature

The shareable brand brief URL. launchkit.wherewithal.studio/dodek as a beautiful, complete, public-facing brand package would simultaneously be the product demo, the onboarding artifact, the press kit, and the design spec. That one feature justifies the entire product. It also directly expresses Inversion #4 — brand identity becomes an API endpoint.

Strategic Alignment

Launchkit's inversion is Infrastructure (#4): make the brand system transparent, composable, and shareable rather than locked in Figma files and Notion docs. The current design doesn't express that inversion. A visitor cannot see the thesis in the UI. The shareable URL concept makes the inversion legible.

Conversion and Trust Signals

No trust signals exist. The product has not earned the right to ask for any effort. Before asking a Founder-Operator to configure anything, the product must show them what a completed configuration looks like. The sample package is the trust signal.

Retention Design

No retention hooks. A returning user has no reason to come back because the product state never changes. The checklist model (Vector 3) would fix this: each visit shows measurable progress toward launch-ready brand packages.

Strategic Gaps (Top 3)

  1. No sample package. Without a live example of Launchkit's output, no persona can understand the product's value or validate its quality.
  2. No shareable URL pattern. The product's primary distribution mechanism — sending a brand brief to an advisor or press contact — doesn't exist at any layer.
  3. No IA decision made. Is this a browser tool, CLI tool, or shareable document? That decision drives every subsequent design choice and hasn't been made after 7 scan cycles.

Mobile Possibilities

The package-as-document format (Vectors 2 and 6) is inherently mobile-friendly — a single-column scrollable document reads well at any viewport. The token browser (Vector 4) needs a card-stack treatment on mobile rather than a grid. The checklist (Vector 3) is excellent on mobile: clear progress bars and tap-to-expand detail.

Mobile Defects

Current public/index.html has no responsive issues because it has almost no layout. The max-width: 1000px container with padding: 2rem renders acceptably at all viewports. No horizontal overflow. No touch target issues — there are no interactive elements.

The Interaction Design Bet

Make the venture selector on the landing page a tap-to-preview interaction: tapping a venture name slides in a preview of their brand brief (right panel on desktop, below on mobile). No page load, no login. That single micro-interaction makes the product feel alive and valuable on first contact.

11. System Consistency

Design Token Audit

No design tokens are defined in the current codebase. The landing page uses raw hex values inline. The three-family font system (Bebas Neue, Barlow Condensed, JetBrains Mono) is a portfolio standard but not loaded or referenced in any Launchkit file.

Component Patterns

No components exist. Single page, no reusable patterns.

Voice and Tone

Current copy: "Launch platform and documentation site for Wherewithal ventures." and "Content coming soon." Tone is neutral-bureaucratic. The Wherewithal portfolio voice is direct, technical, and thesis-forward. Launchkit's copy should match: "One brief. One system. Zero ambiguity." rather than "documentation site."

Brand Fidelity

Dark background and basic layout are consistent with Wherewithal aesthetic. System font stack, absent brand fonts, and placeholder copy make the page feel like scaffolding rather than a portfolio product. Brand fidelity requires: loading three font families, a Launchkit-specific accent color (distinct from all venture colors), and the real positioning statement.

Design Debt

  • All styles inline in a <style> block — no token system. Acceptable now; will need extraction before adding more pages.
  • No CSS custom properties. Raw hex values in all style rules.
  • The three-family font system is a portfolio-wide standard not enforced in any Launchkit file.
